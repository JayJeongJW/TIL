# 1. AI 서치: 게임 데이터 분석의 자동화 시대

## 강연 주제

게임 데이터 분석 서비스에 AI를 결합했을 때 어떤 문제가 생기고, 이를 해결하기 위해 어떤 데이터 구조와 분석 체계가 필요한지를 다룬 강연이다.

핵심 질문은 다음과 같다.

```text
게임 데이터 분석 + AI = 신뢰할 수 있는 자동 분석이 가능한가?
```

기술적으로 AI가 데이터를 읽고 분석하는 것은 가능하다.
하지만 그것이 곧 신뢰할 수 있는 분석 결과를 의미하지는 않는다.

---

## NXLog 리포트란?

NXLog 리포트는 게임과 관련된 지표를 분석할 수 있는 게임 데이터 특화 분석 서비스다.

게임 데이터에는 단순한 숫자 이상의 맥락이 포함되어 있다.

AI가 컬럼 이름과 수치를 읽을 수는 있지만, 그 데이터가 어떤 게임 상황에서 만들어졌는지, 어떤 의도를 가진 지표인지, 어떤 비즈니스 맥락과 연결되는지는 사람이 알려줘야 한다.

```text
AI가 컬럼을 읽는다
≠
AI가 게임의 맥락을 이해한다
```

---

## 사람과 AI의 역할 구분

AI 분석 서비스를 만들기 위해서는 사람이 해야 할 일과 AI가 맡을 수 있는 일을 구분해야 한다.

| 구분 | 역할                         |
| -- | -------------------------- |
| 사람 | 데이터의 의미, 비즈니스 맥락, 분석 기준 정의 |
| AI | 정리된 데이터 기반 탐색, 요약, 분석 보조   |
| 핵심 | 사람과 AI의 경계를 명확히 정하는 것      |

AI를 붙이는 출발점은 “AI가 무엇을 할 수 있는가”가 아니라, **어디까지 AI에게 맡길 수 있고 어디부터 사람이 판단해야 하는가**를 정하는 것이다.

---

## 기존 데이터 분석 방식의 한계

기존 분석 방식에는 크게 세 가지 문제가 있었다.

```text
1. 해석 방식에 대한 가이드라인 부재
2. 대시보드 개발 절차의 복잡성
3. 콘텐츠 과부하와 대시보드 방치
```

---

## 1. 해석 방식에 대한 가이드라인 부재

하나의 대시보드를 보더라도 보는 사람마다 해석이 달라질 수 있다.

```text
하나의 대시보드
→ 수많은 생각
→ 서로 다른 해석
```

데이터를 보여주는 것만으로는 충분하지 않다.
그 데이터를 어떻게 읽어야 하는지, 어떤 기준으로 판단해야 하는지에 대한 가이드가 필요하다.

---

## 2. 대시보드 개발 절차의 복잡성

게임 담당자가 대시보드 개발을 요청하기까지의 절차는 복잡하다.

### 데이터가 이미 있는 경우

```text
요구사항 파악
→ 데이터 확인
→ 대시보드용 데이터 가공
→ 대시보드 개발
→ 검수
→ 배포
```

### 데이터가 없는 경우

```text
요구사항 파악
→ 데이터 설계
→ 로그 개발
→ 로그 QA
→ 로그 배포
→ 데이터 확인
→ 대시보드용 데이터 가공
→ 대시보드 개발
→ 검수
→ 배포
```

데이터가 없다면 로그 설계와 개발, QA, 배포 과정이 추가된다.
즉, 분석 요청 하나가 실제 대시보드로 이어지기까지 많은 단계와 시간이 필요하다.

---

## 3. 콘텐츠 과부하의 위험성

대시보드를 계속 만들다 보면 다음 문제가 생긴다.

```text
대시보드 중복 생산
조회 속도 저하
비용 증가
담당자 변경 시 뷰 재구성
기존 대시보드 방치
```

결과적으로 많은 리소스를 들여 만든 대시보드가 제대로 활용되지 못하고, 또 다른 대시보드를 새로 만드는 악순환이 발생한다.

---

## AI Search 개발 방향

AI Search는 에이전트가 직접 데이터를 분석하는 형태로 접근했다.

이를 위해 넥슨의 데이터 통합 플랫폼인 **모노레이크**가 기반이 되었다.

---

## 모노레이크란?

모노레이크는 넥슨 사내 데이터 통합 플랫폼이다.

```text
데이터 적재
데이터 가공
데이터 분석
데이터 시각화
```

데이터와 관련된 전 영역을 지원하는 종합 플랫폼으로 설명되었다.

강연에서는 이를 “데이터 민주주의를 지향하는 넥슨 사내 데이터 통합 플랫폼”으로 정리했다.

---

## Snowflake Agent Architecture 구조

AI Search는 표준화된 데이터 구조를 기반으로 한다.

이미지 자료에서는 다음과 같은 표준화 데이터 영역이 확인되었다.

```text
KPI Mart
In-game Mart
UP
Community
```

이런 표준화 데이터가 있어야 AI가 안정적으로 분석할 수 있다.

---

## Garbage In, Garbage Out

강연에서 강조된 핵심은 다음 문장이다.

```text
정제되지 않은 데이터로는 양질의 답변이 나올 수 없다.
```

AI 성능이 아무리 좋아도, 입력 데이터가 정리되어 있지 않으면 결과도 신뢰하기 어렵다.

---

## NXLog 표준 로그 규격

NXLog 표준 로그 규격은 fact 중심의 표준화 데이터셋으로 설명되었다.

```text
NXLog 표준 로그 규격
→ fact 중심 표준화 데이터셋
→ AI에게 제공되는 교과서 역할
```

AI에게 좋은 분석을 기대하려면, AI가 읽을 수 있는 좋은 데이터 구조가 먼저 필요하다.

---

## AI Ready Data를 위한 준비

더 좋은 분석 결과를 위해 AI가 읽기 쉬운 형태의 데이터를 제공했다.

| 준비 항목         | 내용                   |
| ------------- | -------------------- |
| 메타데이터 부착      | 데이터 의미와 설명을 함께 제공    |
| 원본 테이블 분할     | AI가 읽기 쉬운 단위로 테이블 정리 |
| 주요 유저 정보 결합   | 분석에 필요한 유저 관련 정보 연결  |
| 사전 Join 관계 구성 | 자주 필요한 관계를 미리 구성     |

---

## Semantic 구성과 Agent 구성

AI Search에서는 Semantic 구성과 Agent 구성이 함께 사용되었다.

| 구분          | 설명        |
| ----------- | --------- |
| Semantic 구성 | 백과사전 형태   |
| Agent 구성    | 분석 가이드 형태 |

Semantic은 데이터의 의미를 알려주는 역할이고, Agent는 분석을 어떻게 진행할지 안내하는 역할에 가깝다.

## AI 분석 서비스의 목표와 한계

AI Search는 대시보드 개발 방식을 바꾸는 시도다.

```text
대시보드 개발자
→ 에이전트 개발자
```

다만 사람이 설계하고 검증한 고품질 대시보드를 완전히 대체하기는 어렵다고 정리했다.

AI는 분석을 보조하고 확장할 수 있지만, 사람이 만든 분석 기준과 검증 과정을 완전히 없애기는 어렵다.

---

## 지금 시작해야 하는 이유

AI 분석 서비스는 한 번에 완성되는 것이 아니라 경험이 쌓이면서 발전한다.

```text
하나의 경험
→ 팀의 자산
→ 회사의 자산
→ 더 나은 분석 플랫폼
```

기술만으로 경쟁하기에는 부족하고, 분석 경험과 지적 자산이 함께 쌓여야 한다.

AI만 사용하는 분석보다, 사람이 쌓아온 대시보드와 분석 경험이 AI와 결합될 때 더 깊은 분석이 가능하다.

# 2. 블루 아카이브 포스트모템 - 게임 디렉터의 관점에서

# 2. 블루 아카이브 포스트모템 - 게임 디렉터의 관점에서

## 강연 주제

**랜덤에서 재미를 추구하면 안 되는 걸까?

* Re: 트라이부터 시작하는 보스전**

블루 아카이브의 보스전 개선 과정을 다룬 포스트모템 강연이다.

발표는 2025년에 진행된 블루 아카이브 총력전 콘텐츠 보스 개선 작업을 중심으로 구성되었다.

핵심 주제는 다음과 같다.

```text
유저가 통제하기 어려운 랜덤 요소가
보스전에서 재미로 작동하는가,
아니면 반복 리트라이와 피로로 작동하는가?
```

---

## 발표의 전체 흐름

```text
블루 아카이브 전투 구조 설명
→ 택틱 중심 전투 문화 설명
→ 보스 개선을 제안하게 된 계기
→ 이차원 게임 전투 트렌드 분석
→ 블루 아카이브 보스 개선 방향 정리
→ 보스별 개선 사례
→ 개선 이후 결과
→ 결론: 전투는 캐릭터의 매력을 보여주는 무대
```

---

## 블루 아카이브의 전투 구조

블루 아카이브의 전투는 모든 행동을 직접 조작하는 방식이 아니다.

학생들은 기본적으로 자동 전투를 진행하고, 유저는 코스트를 사용해 EX 스킬을 선택적으로 사용한다.

| 요소     | 내용                         |
| ------ | -------------------------- |
| 기본 전투  | 학생들이 자동으로 전투 진행            |
| 유저 개입  | 코스트를 사용해 EX 스킬 발동          |
| 공략 방식  | 파티 조합, 스펙, 스킬 순서, 타이밍이 중요  |
| 플레이 문화 | 공략 영상을 보고 택틱을 따라 하는 방식이 형성 |

---

## 블루 아카이브 전투의 특징

발표에서는 블루 아카이브 전투를 다음 구조로 설명했다.

```text
오토 배틀
→ 적과 조우하면 자동으로 전투

카드 게임식 스킬 사용
→ EX 스킬 카드와 코스트를 관리

한정된 자원
→ Cost를 통한 순환 사용
```

블루 아카이브의 전투는 자동 전투와 수동 스킬 사용이 결합된 구조다.
유저가 모든 조작을 직접 수행하지는 않지만, 언제 어떤 스킬을 사용할지는 전투 결과에 큰 영향을 준다.

---

## 택틱 중심 전투

블루 아카이브의 고난도 보스전에서는 **택틱**이 중요한 역할을 한다.

택틱은 특정 파티 조합, 스킬 순서, 스킬 타이밍, 배치 등을 정리한 공략 흐름이다.

```text
동일한 파티 조합
+ 비슷한 스펙
+ 정해진 스킬 순서
+ 정해진 타이밍
= 보스 클리어 가능
```

유저는 공략 영상을 보고 택틱을 따라 하며 보스를 공략한다.

---

## 택틱을 따라 해도 실패하는 이유

문제는 택틱을 잘 따라 했더라도 여러 랜덤 요소 때문에 실패가 발생할 수 있다는 점이다.

발표에서는 전투 중 리트라이를 유발하는 요소를 다음과 같이 정리했다.

```text
패턴
치명타
회피
그로기
```

이 중 하나라도 원하는 흐름대로 나오지 않으면 전투를 다시 시작해야 하는 상황이 발생한다.

---

## 반복 리트라이가 일상화되는 구조

택틱을 성공시키기 위해서는 여러 랜덤 요소를 모두 통과해야 한다.

예시 흐름은 다음과 같다.

```text
원하는 위치에 보스가 등장했는가?
→ 패턴이 유리하게 나왔는가?
→ 유의미한 치명타가 발생했는가?
→ 캐릭터 이동에 문제가 없었는가?
→ 그 외 의도한 흐름대로 진행되었는가?
→ 택틱 성공
```

중간 조건 중 하나라도 어긋나면 리트라이가 발생한다.

이 구조에서는 유저의 조작이나 판단보다, 통제하기 어려운 랜덤이 결과에 영향을 주게 된다.

---

## 보스 개선 제안의 계기

일부 보스는 보유한 랜덤 요소가 많았고, 총력전은 경쟁 콘텐츠이기 때문에 이 랜덤 요소들을 돌파해야 상위 랭크에 기록될 수 있었다.

```text
보스의 랜덤 요소
+ 경쟁 콘텐츠 구조
= 상위 랭크를 위한 반복 리트라이 발생
```

발표에서는 라이브 기간이 길어질수록 전투 콘텐츠의 피로도가 리텐션에 영향을 줄 수 있다고 설명했다.

리텐션 감소는 곧 학생을 새로 확보하려는 비용의 감소로도 이어질 수 있기 때문에, 전투 피로도는 콘텐츠 운영에서 중요한 문제로 다뤄졌다.

---

## 신규 난이도 루나틱 출시와 피로도 문제

신규 난이도 **루나틱**이 출시되면서 기존 난이도보다 더 많은 파티 운용이 필요해졌다.

더 많은 파티 운용은 더 많은 랜덤 요소와 마주한다는 뜻이기도 하다.

```text
더 많은 파티 운용
→ 더 많은 랜덤 요소 발생
→ 더 많은 리트라이
→ 전투 피로도 증가
```

그래서 기존 전투 피로도를 유지한 채 난이도만 확장하기보다, 보스가 보유한 랜덤성을 줄이는 개선이 필요했다.

---

# 이차원 게임 전투 트렌드

## 분석 목적

보스 개선을 진행하기 전에 최근 이차원 게임의 전투 트렌드를 분석했다.

발표에서는 이 분석을 통해 다음 두 가지 방향에 주목했다.

```text
버스트 윈도우
수동적 랜덤의 축소
```

---

## 1. 버스트 윈도우

버스트 윈도우는 짧은 시간 동안 공격 능력을 극대화해 폭발적인 피해를 입히는 전략적 시간대를 의미한다.

```text
제한된 버프 / 디버프 시간
+ 제한된 자원
+ 적절한 스킬 타이밍
= 순간 화력 집중
```

이차원 게임에서는 제한된 시간 안에 버프, 디버프, 딜러의 스킬을 맞춰 사용하는 방식이 중요한 전투 구조로 자리 잡고 있었다.

| 예시 방향    | 내용                     |
| -------- | ---------------------- |
| 명조       | 스왑, 스킬, 캐릭터 교체를 이용한 전투 |
| 젠레스 존 제로 | 패링과 교체를 활용한 전투 흐름      |
| 우마무스메    | 특정 타이밍에 맞춘 선택과 운용      |

버스트 윈도우는 유저가 선택하는 전투 타이밍을 만드는 구조다.

---

## 2. 수동적 랜덤의 축소

이차원 게임에서는 유저가 통제하기 어려운 랜덤 요소를 긍정적인 재미로 만들기 어렵다.

이미 전투를 진행하기 위해 여러 조건을 맞춰야 하는 상황에서, 유저가 제어할 수 없는 랜덤이 추가되면 피로가 증가할 수 있다.

예시로는 다음 상황이 있다.

```text
디버프가 Miss가 되는 경우
그로기가 가능한 공격이 Miss가 되는 경우
공격 준비가 Miss 때문에 무너지는 경우
보스가 갑자기 무적 패턴을 사용하는 경우
```

이런 랜덤 요소는 유저가 준비한 전투 흐름을 끊거나, 택틱 자체를 무효화할 수 있다.

---

## 유저가 이미 랜덤을 겪고 있는 지점

발표에서는 이차원 게임 유저가 전투 이전부터 이미 랜덤을 겪는다고 설명했다.

대표적으로 캐릭터 획득 과정에서 랜덤이 존재한다.

```text
캐릭터 획득
→ 이미 랜덤을 겪고 전투에 진입
```

따라서 전투 안에서까지 과도한 수동적 랜덤이 반복되면 피로가 커질 수 있다.

---

## 블루 아카이브의 기존 랜덤 스탯

블루 아카이브는 이미 다음과 같은 랜덤 관련 스탯을 활용하고 있었다.

```text
명중치
회피치
치명 저항
안정치
보스 패턴
```

다만 이런 스탯은 학생의 능력과도 연결되어 있기 때문에, 학생을 직접 수정하기보다는 보스 쪽을 수정하는 방향으로 접근했다.

---

# 보스 개선 방향

## 전체 개선 방향

분석을 바탕으로 보스 개선은 다음 방향으로 진행되었다.

```text
랜덤 스탯 개선
랜덤 스탯 + 랜덤 패턴 개선
랜덤 패턴 개선
```

보스별 개선 방향은 다음과 같이 정리된다.

| 개선 구분            | 대상 보스            |
| ---------------- | ---------------- |
| 랜덤 스탯 개선         | 그레고리오            |
| 랜덤 스탯 + 랜덤 패턴 개선 | 호드, 고즈, 카이텐저, 비나 |
| 랜덤 패턴 개선         | 쿠로카게, 페로로지라      |

---

## 개선 목표

개선의 주요 목표는 두 가지였다.

```text
리트라이 감소
변하지 않는 공략 유지
```

발표에서는 개선의 주요 목표가 **리트라이를 줄이는 것**과 **기존 공략에서 크게 벗어나지 않는 것**이라고 설명했다.

즉, 보스의 정체성을 완전히 바꾸는 것이 아니라, 유저가 반복 리트라이를 해야 하는 부분을 줄이는 방향에 가까웠다.

---

# 보스별 개선 사례

## 1. 그레고리오

그레고리오는 랜덤 스탯 개선 대상이었다.

개선 방향은 적의 회피 수치를 줄이는 것이었다.

```text
적의 회피 수치 감소
→ 명중하지 않아 리트라이하는 상황 완화
```

그레고리오의 경우 특정 학생이 명중 문제로 활약하지 못하는 상황이 있었고, 회피 수치를 조정해 리트라이 강도를 낮추는 방향으로 수정되었다.

---

## 2. 카이텐저

카이텐저는 랜덤 스탯과 랜덤 패턴이 함께 개선된 보스다.

개선 내용은 다음과 같다.

```text
일부 적의 회피 수치 감소
방패 스킬을 랜덤 타이밍이 아닌 고정 타이밍에 사용하도록 수정
```

기존에는 랜덤 타이밍에 방패 스킬을 사용하면서 전투 흐름이 크게 흔들릴 수 있었다.

개선 후에는 핵심 스킬 사용 타이밍을 고정해 리트라이 요소를 줄였다.

---

## 3. 비나

비나는 명중치, 안정치, 엄폐 관련 요소가 전투 결과에 영향을 주었다.

기존에는 같은 상황에서도 데미지 편차가 발생하고, 엄폐 성공 여부에 따라 엄폐물 파괴와 학생 이동이 달라져 리트라이가 발생할 수 있었다.

개선 방향은 다음과 같다.

```text
명중치 / 안정치 디버프 제거
확률적 엄폐로 인한 리트라이 제거
엄폐 학생이 엄폐 자체로 이점을 얻도록 수정
```

비나는 하위 난이도에서 대부분의 스킬을 확률적으로 사용하던 구조도 있었고, 주요 스킬 타이밍을 고정하는 방향으로 수정되었다.

---

## 4. 호드

호드는 치명 저항과 패턴 관련 리트라이가 문제가 된 보스다.

개선 방향은 다음과 같다.

```text
치명 저항에 영향을 주는 요소 수정
기믹 수행 시 치명 저항 수치 감소
첫 스킬에 따라 전투 흐름이 달라지는 문제 완화
지정된 타이밍에 지정된 스킬 사용
```

기존에는 첫 스킬에 따라 유리한 상황이 달라질 수 있었다.
개선 후에는 스킬 사용 타이밍을 고정해 리트라이 요소를 줄였다.

---

## 5. 고즈

고즈는 보스 개선 발표에서 가장 많이 다뤄진 보스 중 하나다.

기존 고즈는 본체 위치와 스킬 회피 위치가 랜덤하게 결정되는 구조가 있었다.

```text
원하는 위치에 본체가 등장할 때까지 리트라이
스킬을 피하지 않아도 되는 흐름이 나올 때까지 리트라이
그로기 타이밍과 본체 위치가 맞물려 랜덤성 증가
```

개선 방향은 다음과 같다.

```text
본체가 나타나는 위치 순서 고정
확률적 패턴 사용 제거
지정된 타이밍에 지정된 스킬 사용
그로기 누적 방식 변경
그로기 타이밍이 일정해지도록 수정
일부 스킬에 Cost 회복 추가
버스트를 노릴 수 있는 구조 제공
```

기존 마술사 콘셉트를 일부 포기하더라도, 전투 피로도를 줄이는 쪽에 집중했다.

---

## 고즈의 추가 개선

고즈는 본체가 나타나는 위치 순서를 고정하고, 확률적인 패턴 사용을 제거했다.

또한 그로기 누적 방식을 변경해 그로기 타이밍이 일정해지도록 수정했다.

추가로 신규 난이도에서는 그로기의 랜덤성을 제거하고, 신규 패턴은 랜덤 이동 대신 리턴을 추가했다.

일부 스킬에 Cost 회복을 추가해 유저가 버스트 윈도우를 노릴 수 있게 했다.

```text
위치 순서 고정
확률 패턴 제거
그로기 타이밍 안정화
Cost 회복 추가
버스트 윈도우 제공
```

---

## 6. 페로로지라

페로로지라는 학생 이동과 전투 환경으로 인해 리트라이가 발생하는 문제가 있었다.

기존에는 복잡한 패턴 자체보다, 전투 환경으로 인해 학생들의 이동이 랜덤하게 꼬이는 문제가 컸다.

개선 방향은 다음과 같다.

```text
확률적 이동 문제 완화
전투 환경으로 인한 리트라이 요소 감소
기존 활용성을 일부 포기하더라도 전투 피로도 감소에 집중
```

발표에서는 편한 패턴은 아니었지만, 전투 환경으로 인해 학생들이 의도하지 않게 이동하며 문제가 생기는 보스였다고 정리했다.

---

## 7. 쿠로카게

쿠로카게는 랜덤 패턴과 대상 선택 구조가 문제로 다뤄졌다.

기존에는 초음파와 관련된 패턴에서 대상 지정 방식이 랜덤하게 느껴질 수 있었고, 유저가 해법을 알아도 실제 전투 흐름이 꼬일 수 있었다.

개선 방향은 다음과 같다.

```text
대상 선택 규칙 정리
대상 인덱스 수정
정확히 설명을 적용하는 방식으로 타겟팅 개선
랜덤하게 느껴지는 패턴 해소
```

쿠로카게는 단순히 랜덤을 줄이는 것뿐 아니라, 유저가 패턴을 이해하고 대응할 수 있도록 구조를 정리하는 방향으로 개선되었다.

---

# 개선 결과

## 커뮤니티 반응

개선 이후 커뮤니티에서는 긍정적인 반응이 관측되었다.

특히 고즈와 그레고리오 관련 개선은 유저가 바로 체감할 수 있는 형태였다.

```text
패턴 고정
회피 정상화
리트라이 감소
개발팀이 문제를 인지하고 있다는 반응
```

발표에서는 개선 이후 전투 피로도와 관련한 긍정적인 동향이 관측되었다고 정리했다.

또한 개발팀에 대한 신뢰 증가와 개선 방향에 대한 긍정적인 분위기가 형성되었다.

---

## 실제 데이터 결과

실제 플레이 데이터에서도 평균 클리어 타임과 전투 실패 비율이 유의미하게 변화했다.

### 고즈

| 구분        |    인세인 |    토먼트 |
| --------- | -----: | -----: |
| 평균 클리어 타임 | 20% 감소 | 34% 감소 |
| 전투 실패 비율  | 31% 감소 | 42% 감소 |

### 쿠로카게

| 구분        |    인세인 |    토먼트 |
| --------- | -----: | -----: |
| 평균 클리어 타임 | 17% 감소 | 24% 감소 |
| 전투 실패 비율  | 27% 감소 | 34% 감소 |

리트라이가 줄어든 만큼 평균 클리어 타임과 전투 실패 비율이 함께 감소한 것으로 정리된다.

---

## 상위 난이도 트로피 추이

보스 개선 전후로 상위 난이도 트로피 플레이 비율도 확인되었다.

```text
고즈 개선 전: 4.5%
고즈 개선 후: 8.4%

쿠로카게 개선 전: 7.1%
쿠로카게 개선 후: 10.6%
```

다만 발표에서는 이미 설정되어 있는 보스의 랜덤을 제거한다는 것은 난이도를 낮추는 조정으로 보일 수 있다고 설명했다.

장기적으로 전투 피로도를 해결하려면, 기존 보스 조정뿐 아니라 신규 난이도와 신규 보스 설계 단계에서도 더 신중한 접근이 필요하다고 정리했다.

---

# 개선의 성공과 남은 문제

## 성공한 부분

```text
리트라이 감소
평균 클리어 타임 감소
전투 실패 비율 감소
유저 체감 개선
개발팀에 대한 신뢰 증가
```

보스 개선은 반복 리트라이를 줄이고, 전투 피로도와 관련된 문제를 완화하는 데 효과가 있었다.

---

## 남은 문제

보스 개선이 모든 문제를 해결한 것은 아니었다.

```text
기존 공략과 충돌하는 요소 발생 가능
신규 난이도에서 전투 피로도 재발 가능
보스의 기억과 이해 난이도 문제
경쟁 콘텐츠 특성상 피로도 완전 제거는 어려움
```

예를 들어 신규 난이도에서는 기존과 다른 구조가 추가되면서 새로운 리트라이 요소가 생길 수 있다.

또한 기존 공략과 충돌하는 사유로 인해 남아 있는 랜덤 요소도 계속 관리해야 하는 과제로 남았다.

---

# 결론: 전투는 캐릭터의 매력을 보여주는 무대

## 전투의 역할

발표의 결론은 다음 문장으로 정리된다.

```text
전투는 캐릭터의 매력을 보여주는 무대다.
```

블루 아카이브 같은 이차원 게임에서 전투는 단순히 승패를 가르는 콘텐츠가 아니다.

유저가 획득하고 육성한 캐릭터가 활약하는 공간이다.

따라서 전투가 스트레스를 부여하는 장소가 되어서는 안 되고, 캐릭터의 매력을 보여주는 무대처럼 동작해야 한다.

---

## 이차원 게임 전투의 방향

발표에서는 최근 이차원 게임 전투의 방향을 다음과 같이 정리했다.

```text
사라져가는 Miss
정교해지는 버스트 윈도우
랜덤 패턴의 축소
캐릭터와의 교감 유지
```

근래의 이차원 게임은 캐릭터와의 교감을 방해하지 않는 전투를 지향하고 있다.

기믹이 너무 복잡해 유저에게 스트레스를 주지 않도록 하기 위한 많은 고려가 필요하다고 설명했다.

---

## 랜덤에 대한 결론

발표의 핵심 질문은 다음과 같다.

```text
랜덤에서 재미를 추구하면 안 되는 걸까?
```

결론은 랜덤 자체가 항상 나쁜 것은 아니지만, 유저가 통제할 수 없는 수동적 랜덤은 이차원 게임 전투에서 피로로 작동하기 쉽다는 것이다.

| 구분      | 설명                                    |
| ------- | ------------------------------------- |
| 수동적 랜덤  | 회피율, Miss, 보스 랜덤 패턴처럼 유저가 통제하기 어려운 랜덤 |
| 능동적 플레이 | 유저가 예측하고 대응하며 전투를 풀어가는 플레이            |
| 개선 방향   | 보이지 않는 불쾌한 랜덤을 줄이고, 대응 가능한 전투 흐름을 제공  |

---

## 블루 아카이브 보스 개선 핵심 정리

```text
문제
→ 택틱을 따라 해도 랜덤 요소 때문에 반복 리트라이가 발생함
→ 경쟁 콘텐츠에서는 랜덤을 돌파해야 상위 랭크를 노릴 수 있음
→ 랜덤이 재미보다 피로와 스트레스로 작동하는 경우가 있었음

분석
→ 최근 이차원 게임 전투 트렌드 확인
→ 버스트 윈도우와 수동적 랜덤 축소에 주목
→ 블루 아카이브의 명중, 회피, 치명 저항, 보스 패턴을 점검

개선
→ 회피 수치 감소
→ 치명 저항 수치 감소
→ 확률적 패턴 사용 제거
→ 핵심 스킬 타이밍 고정
→ 그로기 구조 조정
→ 보스별 랜덤 패턴 개선

결과
→ 평균 클리어 타임 감소
→ 전투 실패 비율 감소
→ 상위 난이도 플레이 비율 변화
→ 전투 피로도 관련 긍정 반응 확인

결론
→ 전투는 캐릭터의 매력을 보여주는 무대
→ 유저가 통제할 수 없는 랜덤은 피로로 이어질 수 있음
→ 랜덤을 줄이고, 대응 가능한 전투 흐름을 만드는 방향으로 개선
```

# 3. 넥슨과 크래프톤의 AX 여정

## 강연 주제

넥슨과 크래프톤이 AI Transformation, 즉 AX를 어떻게 추진하고 있는지에 대한 대담형 강연이다.

주요 내용은 다음과 같다.

```text
AX 도입 방식
조직별 확산 방식
AI 비용과 토큰 사용
AI Ready Data와 온톨로지
AI 네이티브 워크플로우
게임 산업에서 사람의 역할
```

---

## 넥슨의 AX 초기 방향

넥슨은 AX 초기 단계에서 손이 많이 가는 업무의 효율화에 집중했다.

```text
운영 부담 줄이기
라이브 서비스 대응 역량 늘리기
개발 생산량 향상
개발 파이프라인 설계, 다양화, 혁신 추진
```

넥슨은 조직 규모가 크기 때문에 하나의 표준 방식으로 AX를 추진하기 어려웠다.

그래서 각 조직별로 성공한 사례를 전파하는 방식이 사용되었다.

```text
바텀업 방식
→ 다양한 시도
→ 성과를 낸 조직 사례 전파
```

---

## 크래프톤의 AX 초기 방향

크래프톤은 문제 해결 방식에서 **AI First**를 선언했다.

```text
AI에게 먼저 사용해보기
AI를 먼저 도입하기
내가 하고 있는 단위 업무를 AI로 측정해보기
```

AI First는 단순 도구 도입이 아니라, 업무를 시작할 때 AI를 먼저 고려하는 운영 방식으로 설명되었다.

---

## 운영 콘솔과 업무 도구 변화

AX가 가장 체감된 부분 중 하나는 운영 콘솔과 운영 도구의 변화였다.

기존에는 운영 툴이 필요할 때마다 개발 요청을 해야 했고, 개발 인력이 부족하면 대응이 늦어졌다.

```text
기존 방식
운영 툴 필요
→ 개발 요청
→ 긴 대기 시간
→ 운영 대응 지연
```

AX 이후에는 데이터가 필요한 조직이 직접 데이터를 추출하고 확인할 수 있는 기반이 마련되었다.

```text
데이터 직접 추출
자체 모니터링 도구 제작
운영 도구 직접 제작
라이브 대응 민첩성 증가
```

---

## 실패를 바라보는 방식

크래프톤은 실패 징후를 판단하는 별도 지표를 만들기보다, 시행착오 자체를 경험으로 보았다.

```text
실패 지표보다 중요한 것
→ 다양한 시도
→ 시행착오
→ 경험의 자산화
```

AI 도구를 만들 수 있더라도 실제 업무에서 사용하지 못하면 실패로 본다.
다만 그 실패를 끝으로 보지 않고, 어떻게 해결해 성공 사례로 만들 것인지 고민하는 흐름이 강조되었다.

---

## 넥슨의 자체 AI 환경 고민

넥슨은 외부 AI 도구를 도입하는 과정에서 보안과 운영 비용 문제를 겪었다.

```text
외부 Claude 사용의 보안 문제
자체 NX Claude 개발
운영 비용 증가
인프라 비용 부담
전사 도입 시점에 대한 고민
```

외부 도구를 따라가지 않기도 어렵고, 그대로 따라가기에는 운영비와 인프라 비용이 크다는 문제가 있었다.

---

## 빠른 인터랙션과 브릿지 역할

AX 추진 과정에서는 빠른 인터랙션이 중요하다고 정리되었다.

실패했을 때 “이렇게 했으니 안 된다”로 끝내는 것이 아니라, 먼저 시도하고 있는 조직과 지원이 필요한 조직을 연결하는 중간 브릿지 역할이 필요했다.

```text
먼저 시행한 조직
→ 경험 보유
→ 지원이 필요한 조직과 연결
→ AX 확산
```

---

## AX 확산의 허들

AX 과정에서 가장 큰 허들 중 하나는 성공 사례를 공유하는 것이었다.

AI를 잘 사용한 사례가 있어도, 다른 조직이 “저건 나와 상관없는 이야기”라고 느끼면 확산되지 않는다.

이를 해결하기 위해 크래프톤에서는 FD 포지션과 AX 포털이 언급되었다.

---

## FD 포지션

FD는 현업 중심 AX를 지원하는 역할로 설명되었다.

```text
기술 문제 지원
인프라 문제 지원
방법론 지원
현업에 파견되어 문제 해결
```

FD가 있는 자리에는 “AI를 잘 쓰시는 분입니다. 다양한 걸 질문해 주세요”라는 식의 안내가 있었고, 구성원들이 자연스럽게 질문할 수 있는 환경을 만들었다.

예를 들어 엑셀 작업을 AI로 처리하고 싶은 구성원에게 적합한 툴과 방법을 안내하는 방식이다.

---

## AX 포털

크래프톤은 AX 과정에서 나온 결과물을 모아두는 별도 포털을 만들었다.

```text
AX 포털
→ 사내 결과물 저장
→ 앱스토어처럼 접근
→ 권한이 있으면 사용 가능
→ 향후 마켓플레이스 형태로 확장 계획
```

AX 포털은 사내에서 만들어진 AI 활용 결과물을 더 쉽게 공유하기 위한 구조다.

---

## AI 기술 발전 속도와 피로감

AI 기술 발전 속도가 빠르기 때문에, 조직 구성원들은 피로감을 느낄 수 있다.

넥슨은 이 피로감을 줄이기 위해 효용감과 장기 로드맵을 중요하게 봤다.

```text
AI 변화 속도 증가
→ 구성원 피로감 발생
→ 업무가 쉬워지고 빨라지는 효용감 필요
→ 장기 AX 로드맵 제시 필요
```

장기 로드맵은 빠르게 변하는 AI 환경 속에서도 구성원이 3개월, 6개월 뒤를 예측할 수 있게 하는 역할을 한다.

---

## 크래프톤의 접근: 시행착오 기반

크래프톤은 AI를 선도하는 기업이 국내에 많지 않다는 점을 인정하고, 새로운 모델과 툴이 나올 때마다 다양한 시도를 하는 것이 중요하다고 보았다.

```text
아무것도 하지 않으면 아무것도 얻지 못함
→ 다양한 시도 필요
→ 시행착오 경험 필요
→ 경험 전파가 중요
```

특정 방법론을 중앙에서 하나로 정하기보다, 현업이 직접 경험하고 판단할 수 있는 기반을 제공하는 방식이 강조되었다.

---

## 게임 산업과 AX의 차이

게임 산업은 다른 산업과 다르게 “재미”를 만드는 산업이다.

문제는 재미에는 정답이 없다는 점이다.

그래서 AX를 적용하기 쉬운 영역과 어려운 영역이 명확히 나뉜다.

| 영역            | 특징                           |
| ------------- | ---------------------------- |
| AX 적용이 쉬운 영역  | 자동화, 운영, 반복 업무, 효과 검증이 쉬운 업무 |
| AX 적용이 어려운 영역 | 재미, 창의성, 판단, 감성, 불확실성이 큰 업무  |

현재는 효과를 검증하기 쉬운 영역부터 AX가 적용되고 있다.
하지만 앞으로는 창의성과 판단이 중요한 영역도 마주하게 될 수밖에 없다.

---

## 사람이 반드시 해야 하는 일

게임 산업에서 AX의 핵심 질문은 다음과 같이 바뀐다.

```text
AI가 무엇을 대신할 수 있는가?
→ 사람이 반드시 무엇을 해야 하는가?
```

게임 산업에서 AX는 단순히 사람의 업무를 AI가 대체하는 과정이 아니라, 전체 워크플로우 안에서 사람이 어떤 일을 해야 하는지 다시 정의하는 과정으로 설명되었다.

---

## 크래프톤이 본 게임 산업의 AX 특징

크래프톤은 게임 산업이 커머스나 금융권보다 AI 도입이 더 용이한 산업이라고 설명했다.

커머스나 금융은 거버넌스와 규제 이슈가 강하지만, 게임 산업은 크리에이티브가 중심이기 때문에 AI를 활용해 제작 과정을 더 확장할 가능성이 크다고 보았다.

```text
반복 제작 작업을 AI로 줄임
→ 절약한 시간을 더 좋은 에셋, 사운드, 표현 방식에 투자
→ 크리에이티브 확장
```

---

## AI Ready Data와 온톨로지

게임에는 문서, 소스 코드, 리소스, 플레이 데이터 등 맥락 데이터가 많다.

이를 AI가 활용하려면 AI Ready Data로 정리해야 한다.

넥슨은 방대한 유저 플레이 데이터를 오래전부터 모아왔고, 앞으로는 AI가 데이터를 잘 활용할 수 있도록 데이터의 위치와 관계를 명확히 기록하는 데 집중하고 있다고 설명했다.

```text
문서
데이터
유저 플레이 데이터
기획서
소스 코드
리소스
```

이 데이터들이 서로 어떤 관계를 갖는지 온톨로지화하는 것이 중요하다.

---

## 모든 게임 데이터를 하나로 표준화하기 어려움

넥슨은 서비스하는 게임이 많기 때문에 모든 게임 데이터를 하나의 방식으로 표준화하는 것은 어렵다고 보았다.

대신 다양한 게임 데이터를 유연하게 받아들일 수 있는 구조와, 각 데이터가 어디에 있는지 설명하는 온톨로지가 필요하다고 정리했다.

---

## 크래프톤의 데이터 파운데이션

크래프톤은 게임 제작 관련 데이터 에셋과 비게임 제작 관련 에셋을 나누고, 온톨로지 기반 카탈로그를 만들었다.

```text
게임 제작 에셋
비게임 제작 에셋
온톨로지 기반 카탈로그
AI가 접근 가능한 구조
```

글로벌 게임사이기 때문에 유럽, 한국, 일본, 중국 등 각 지역의 AI 관련 법과 기준도 고려했다.

이를 바탕으로 다음 영역을 구분했다.

```text
시도해볼 수 있는 영역
실제 제작에 활용할 수 있는 영역
퍼블리싱 가능한 영역
```

---

## AI 비용과 토큰 비용

AI 도입에서 가장 큰 이슈 중 하나는 비용이다.

넥슨은 초기 바텀업 확산을 위해 다양한 AI 도구를 적극 지원했지만, 비용이 예상보다 빠르게 증가했다.

중요한 것은 비용 자체보다, 그 비용이 얼마나 가치 있는 업무로 연결되는지를 측정하는 것이다.

```text
토큰 사용량
활용 패턴
프로젝트 진행 상황
생산성 향상 여부
콘텐츠 생산 증가
유저 불편 해소
```

비용을 줄이는 것만이 목적이 아니라, 더 큰 가치로 연결되는 사용인지 판단하는 것이 핵심이다.

---

## 크래프톤의 비용 대시보드

크래프톤은 AI 비용을 세 가지로 나눠 대시보드로 구성했다.

| 비용 구분        | 내용                    |
| ------------ | --------------------- |
| 토큰 비용        | LLM 사용량 기반 비용         |
| 엔터프라이즈 시트 비용 | 조직 단위 AI 툴 사용 비용      |
| 개인 구독형 지원 비용 | 개인이 사용하는 구독형 도구 지원 비용 |

비용 대시보드를 공개하면 서로 다른 질문이 동시에 나온다.

```text
비용을 줄이라는 뜻인가?
토큰을 많이 쓰라는 뜻인가?
```

크래프톤은 조직의 AI 리터러시 성숙도에 따라 가이드를 다르게 줘야 한다고 설명했다.

```text
AI 리터러시가 낮은 조직
→ 토큰 사용 장려

성숙도가 높은 조직
→ 비용 효율 가이드 필요
```

---

## 모델 선택과 가성비

크래프톤은 LLM 모델을 선택할 때 참고할 수 있는 단가표와 가성비 지표를 만들었다.

```text
GPT-4o
Opus 3.5
기타 모델
```

각 모델의 성능과 비용을 비교해 구성원이 상황에 맞게 선택할 수 있도록 한 것이다.

---

## 토큰 비용 예측의 어려움

AI 모델의 가격 정책은 외부 업체의 결정에 따라 변할 수 있다.

넥슨은 하반기 비용을 예측하기 위해 비용 예측 모델을 만들고 있지만, 데이터가 조금만 바뀌어도 결과가 달라지는 어려움이 있다고 설명했다.

```text
오늘 데이터로 예측한 결과
≠
내일 데이터로 예측한 결과
```

그래도 전체적인 스케일 감을 맞추기 위해 계속 비용 시뮬레이션 모델을 연구하고 있다.

---

## AI 네이티브 워크플로우

AX가 더 깊게 들어가면 기존 워크플로우를 그대로 둔 채 일부만 AI로 바꾸는 수준을 넘어서야 한다.

넥슨은 기존 워크플로우를 AI가 활용할 수 있는 형태로 정리하는 데 집중하고 있다고 설명했다.

```text
업무 세분화
업무 표준화
각 단위를 에이전트가 수행할 수 있도록 구성
AI 네이티브 워크플로우로 전환
```

---

## AX를 효율화 도구로만 보면 생기는 한계

AX를 단순 효율화 도구로만 보면 같은 콘텐츠를 두 배, 세 배 더 찍어내는 수준에서 끝날 수 있다.

중요한 것은 시간을 줄이는 것뿐 아니라, 사람이 무엇에 집중해야 하는지 다시 정의하는 것이다.

```text
게임 개발 조직
→ 더 재미있는 게임을 만드는 일에 집중

CS 조직
→ 유저 불편을 더 빠르고 정확하게 해소하는 일에 집중
```

단순히 빠르게 답변하는 것이 좋은 CS가 아니다.
업무의 본질이 무엇인지 파악하고, AI를 통해 그 본질을 더 잘 수행하는 것이 중요하다고 정리되었다.

---

## 크래프톤의 워크와 워크플로우 구분

크래프톤은 업무를 크게 Work와 Workflow로 나누어 고민했다.

```text
Work
→ 단위 업무

Workflow
→ Work들이 연결된 흐름
```

현재는 전사 전체를 한 번에 연결하기보다, 단위 조직의 업무 자동화를 하나의 프로세스로 만드는 경험을 쌓고 있다.

예시로는 Slack을 통해 들어오는 요청을 AI로 처리하고, 다시 Slack으로 결과를 회신하는 흐름 전체를 자동화하는 방식이 언급되었다.

---

## 넥슨과 크래프톤이 서로 참고한 부분

두 회사는 비슷한 고민을 하고 있다는 점을 확인했다.

크래프톤은 넥슨과 같은 큰 게임사가 비슷한 AX 문제를 겪고 있다는 점에서 동료를 만난 느낌이라고 말했다.

넥슨은 크래프톤이 조직별로 토큰 사용 방식을 나누는 접근을 인상 깊게 보았다.

```text
AX를 위해 토큰을 많이 써야 한다
vs
비용이 너무 많이 나오므로 줄여야 한다
```

이 두 생각이 충돌하는 상황에서, 조직의 성숙도에 따라 다른 기준을 적용하는 방식이 힌트가 되었다.

---

## AX 진척도 평가

넥슨은 현재 업무가 얼마나 구체화되었고 에이전트화되었는지를 중심으로 AX 진척도를 평가하고 있다.

```text
업무를 AI가 수행할 수 있는 단위로 분해했는가?
실제로 자동화했는가?
에이전트화했는가?
```

다만 이는 사전 작업에 가까우며, 장기적으로는 AI 네이티브 워크플로우 전환과 실제 시간·비용 절감 효과로 이어질 수 있다고 보았다.

더 장기적으로는 효율성뿐 아니라, 이전에는 불가능했던 가치를 창출했는지도 평가해야 한다고 정리했다.

---

## 크래프톤의 AX 평가 방식

크래프톤은 평가 기준을 세우는 것 자체를 조심스럽게 바라보았다.

AI를 잘 쓰면 시간이 남아야 하는 것처럼 오해할 수 있고, 정성적 평가에는 모호함이 있기 때문이다.

그래서 성공 사례를 등록하고 공유하는 교류 자체, 시행착오를 서로 알게 만드는 구조를 중요하게 보았다.

```text
내가 모르는 것을 누구에게 물어볼 수 있는가
모르는 것을 설명할 수 있는가
시행착오가 조직의 자산이 되는가
```

이런 흐름을 AX가 자산화되고 있다는 신호로 보았다.

## 주니어에게 남긴 메시지

AI가 빠르게 발전하고 있어 실무자들도 기대감과 두려움을 동시에 느낀다고 했다.

게임 업계는 항상 변화와 함께 성장해왔고, 플랫폼과 개발 방식이 바뀔 때마다 새로운 기회가 열렸다.

```text
AI에 대한 두려움보다 기대감에 무게를 두기

```

기존 고정관념이 적고, 처음부터 AI를 활용하며 일하게 될 세대이기 때문이다.

```text
AI를 쓰는 것이 자연스러운 세대

```

# 4. 연결의 시대, 게임은 어디로 가는가?

## 강연 주제

**연결의 시대, 게임은 어디로 가는가?**

게임, 크리에이터, 플랫폼, 커뮤니티, IP, AI가 연결되는 시대에 게임이 어디로 이동하고 있는지를 다룬 대담이다.

핵심 주제는 다음과 같다.

```text
시간 경제
관심 경제
크리에이터와 게임사의 관계
라이브 게임과 커뮤니티
IP의 횡적 확장
AI 시대의 게임과 크리에이터
신뢰와 맥락
```

---

## 시간 경제와 관심 경제

강연에서는 사람들이 무엇에 시간을 쓰는지가 중요해졌다는 점이 먼저 다뤄졌다.

게이머들이 게임을 즐기는 시간이 줄어들고 있다면, 그 시간이 어디로 이동했는지를 알아야 한다.

```text
게이머의 시간이 줄어든다
→ 그 시간은 어디로 이동했는가?
→ 어떤 콘텐츠에 열광하는가?
→ 게임은 어떤 요소를 다시 넣어야 하는가?
```

게임은 K-POP, 숏폼, 스트리밍, OTT 등 다른 엔터테인먼트 콘텐츠와 유저의 시간을 두고 경쟁한다.

---

## 게임이 주던 성취감의 변화

과거 게임은 일상에서 얻기 어려운 성취감을 비교적 쉽게 제공했다.

```text
현실의 성취
→ 많은 시간, 노력, 경쟁 필요

게임의 성취
→ 동등한 출발점
→ 비교적 짧은 시간
→ 명확한 보상과 성장
```

하지만 이제는 게임보다 더 쉽고 가볍게 도파민과 반응을 주는 콘텐츠가 많아졌다.

숏폼, 릴스, 쇼츠, 넷플릭스 같은 매체가 유저의 시간을 가져가고 있다.

---

## 라이브 게임과 커뮤니티

넥슨은 라이브 게임을 오래 서비스하는 회사로 언급되었다.

과거 패키지 게임은 완성된 게임을 출시하고, 유저가 구매해 플레이한 뒤 끝나는 구조였다.

하지만 라이브 게임은 다르다.

```text
게임사
유저
크리에이터
플랫폼
커뮤니티
```

이 여러 주체가 결합해 하나의 커뮤니티를 만든다.

메이플스토리, FC 온라인처럼 오래 서비스되는 게임은 유저에게 소속감을 제공하고, 하나의 생활권처럼 작동할 수 있다.

---

## 라이브 게임 시대에 달라진 역량

예전에는 업데이트를 잘 만들고 서비스하는 것까지가 중요했다.

지금은 그것을 어떻게 보여주고, 어떤 의도를 가지고 업데이트했는지 설명하며, 앞으로 어떤 로드맵으로 운영할 것인지 전달하는 것이 중요해졌다.

```text
업데이트를 잘 만든다
+
그 의도를 잘 설명한다
+
로드맵을 공유한다
+
유저가 계속 머물 이유를 만든다
```

유저가 “이 게임을 계속할 만하다”고 느끼려면, 게임사가 어떤 방향으로 게임을 운영하는지 이해할 수 있어야 한다.

---

## 크리에이터는 동반자

강연에서는 크리에이터를 단순 홍보 수단이 아니라 동반자로 보았다.

사람들은 반드시 옳은 말만 따르지 않는다.
자신이 좋아하는 사람, 믿는 사람, 가까이 느끼는 사람의 말을 듣고 신뢰한다.

크리에이터는 유저가 모르는 정보를 전달하고, 정보 비대칭을 해소하는 역할을 한다.

```text
크리에이터
→ 유저가 좋아하고 신뢰하는 사람
→ 정보 비대칭 해소
→ 게임과 유저를 연결하는 스피커
```

---

## 게임사와 크리에이터의 관계 변화

과거에는 게임 방송이나 크리에이터의 영향력이 크지 않았다.

하지만 지금은 게임사 고위 관계자가 크리에이터 채널에 출연해 더 솔직한 이야기를 할 수 있는 시대가 되었다.

언론에서는 말하기 어려운 표현이나 분위기를 크리에이터 채널에서는 더 자연스럽게 전달할 수 있다.

이 변화는 크리에이터의 위상이 크게 올라갔음을 보여준다.

---

## 데이브 더 다이버 사례

데이브 더 다이버는 출시 전 크리에이터 인터뷰를 통해 긍정적인 기대감을 만들었다.

게임 출시 전임에도 시청자 반응이 좋았고, 이후 게임이 실제로 좋은 평가를 받으면서 신뢰가 이어졌다.

또한 게임 대상과 관련된 크리에이터 채널의 설문 결과가 공식 석상에서 언급될 정도로, 크리에이터 커뮤니티의 의견이 의미 있는 지표로 다뤄졌다.

---

## 스트리머가 게임 흥행에 기여한 사례

강연에서는 스트리머와 크리에이터가 게임 흥행에 영향을 준 사례들이 언급되었다.

```text
항아리 게임
수박 게임
어몽어스
Slay the Spire
하스스톤
발로란트
배틀그라운드 멸망전
메이플랜드
팀파이트 매니저
```

수박 게임은 일본 버튜버들이 플레이하면서 큰 인기를 얻었고, 어몽어스는 스트리머 플레이를 통해 폭발적으로 확산되었다.

Slay the Spire도 스트리머의 플레이가 큰 전환점이 된 사례로 언급되었다.

---

## 게임 정보 유튜버의 역할

크리에이터의 역할은 단순 플레이 방송에만 있지 않다.

게임 정보 유튜버는 게임 관련 이슈를 설명하고, 인디 게임을 소개하고, 정책적 문제를 다루는 방식으로도 기여한다.

예시로 GTA RP 서버와 관련된 법적 이슈, 메이플랜드와 메이플스토리 월드, 인디 게임 홍보 사례 등이 언급되었다.

---

## 광고와 신뢰

크리에이터가 게임 광고를 할 때 중요한 것은 맹목적인 칭찬이 아니다.

```text
깔 것은 깐다
좋은 것은 좋다고 말한다
단점도 함께 설명한다
취향에 맞는 유저에게 추천한다
```

이런 방식으로 시청자와의 신뢰를 유지할 때 광고 효과도 높아질 수 있다.

ROAS, CPA 같은 지표도 언급되었고, 크리에이터가 게임에 얼마나 영향을 주었는지를 측정하는 방식도 고도화되고 있다고 설명되었다.

---

## 보는 게임과 하는 게임

게임은 더 이상 직접 플레이해야만 소비되는 콘텐츠가 아니다.

```text
직접 플레이하는 게임
보는 게임
자동 사냥처럼 간접적으로 즐기는 게임
커뮤니티로 연결되는 게임
```

게임을 직접 하지 않아도 방송을 보고 정보를 알고 있으면, 나중에 게임에 진입할 때 진입장벽이 낮아진다.

크리에이터는 보는 게임과 하는 게임 사이를 연결하는 역할을 한다.

---

## 퍼널과 측정

플랫폼이 발달하면서 크리에이터 콘텐츠에서 게임으로 넘어오는 흐름을 측정할 수 있게 되었다.

```text
크리에이터 콘텐츠 시청
→ 링크 클릭
→ 게임 진입
→ 플레이
→ 결제 또는 활동
```

이 흐름은 ROAS, CPA 같은 지표로 측정될 수 있다.

게임사는 크리에이터가 게임에 어떤 영향을 주는지 더 정교하게 분석하기 시작했다.

---

## IP의 횡적 확장

게임은 하나의 IP다.

IP는 게임 안에만 머무르지 않는다.

```text
굿즈
공연
오케스트라
오프라인 행사
코스프레
커뮤니티 콘텐츠
2차 창작
```

K-POP IP가 굿즈, 포토카드, 공연으로 확장되는 것처럼, 게임 IP도 다양한 방향으로 확장될 수 있다.

게임 IP는 디지털 콘텐츠와 굿즈 측면에서 확장 가능성이 크다.

넥슨도 최근 IP의 횡적 확장을 중요하게 보고 있다고 정리되었다.

---

## 크리에이터와 게임사의 공생

크리에이터는 게임사가 없으면 콘텐츠 소재를 잃고, 게임사는 크리에이터를 통해 유저와 연결될 수 있다.

```text
게임사
→ 좋은 게임과 IP 제공

크리에이터
→ 정보를 전달하고 관심을 확산

유저
→ 콘텐츠를 보고 게임에 연결
```

따라서 게임사와 크리에이터는 함께 성장해야 하는 관계로 정리되었다.

---

## AI 시대와 게임의 경쟁

AI는 게임과도 경쟁할 수 있다.

AI는 즉각적인 반응을 제공하고, 사용자가 원하는 답변을 빠르게 준다.

게임이 가진 강점 중 하나가 양방향성과 반응성이었는데, AI도 그 욕구를 일부 채울 수 있게 되었다.

하지만 AI의 반응은 시간이 지나면 아첨 편향처럼 느껴질 수 있고, 게임이 제공하는 도전과 성취감과는 다르다.

```text
AI가 주는 만족
→ 빠른 반응
→ 칭찬
→ 즉각성

게임이 주는 만족
→ 도전
→ 실패
→ 극복
→ 손떨리는 성취감
```

---

## AI와 제작자의 변화

AI는 게임 제작의 진입장벽을 낮춘다.

누구나 무언가를 만들 수 있는 시대가 되지만, 누구나 잘 만들 수 있다는 뜻은 아니다.

```text
AI로 60점짜리 결과물은 쉽게 만들 수 있음
하지만 80점, 90점짜리 게임은 여전히 기획력, 제작 지식, 감각이 필요함
```

AI는 혼자 만들 수 있는 범위를 넓히고, 한쪽 능력만 가진 사람도 다른 영역을 보완해 게임을 만들 수 있게 한다.

---

## 큐레이션의 중요성

AI로 콘텐츠와 게임이 넘쳐나면, 유저는 무엇을 선택해야 할지 더 어려워진다.

넷플릭스에서 스크롤만 하다가 끄는 것처럼, 게임도 선택 피로가 커질 수 있다.

그래서 앞으로는 믿을 수 있는 추천과 큐레이션이 중요해진다.

```text
너무 많은 게임
→ 선택 피로
→ 믿을 수 있는 추천 필요
→ 크리에이터의 안목과 신뢰 중요
```

크리에이터는 단순 광고자가 아니라, 자신의 안목을 인정받는 큐레이터 역할을 하게 된다.

---

## 크리에이터의 영역 확장

크리에이터는 영상 제작자나 방송인에만 머무르지 않을 수 있다.

AI로 게임 개발 허들이 낮아지면, 게임 개발자도 유튜버처럼 빠른 주기로 게임을 만들고 공개하는 시대가 올 수 있다.

강연에서는 MCN과 유사한 개념으로 MDN이 언급되었다.

```text
MCN: Multi-Channel Network
MDN: Multi-Developer Network
```

MDN은 게임 개발자들이 게임을 빠르게 출시하고 운영할 수 있도록 행정, 배포, 업데이트, 세금 처리 등을 지원하는 구조로 설명되었다.

---

## 작은 게임과 큰 게임 모두 AI의 영향을 받는다

AI는 작은 게임과 큰 게임 모두에 영향을 준다.

작은 게임은 더 빠르게 만들 수 있고, 큰 게임은 더 빠른 제작과 검수, 리소스 생산을 시도할 수 있다.

다만 AI 생성물 활용에는 무단 학습, 저작권, 검수 문제도 함께 따라온다.

```text
작은 게임
→ 소수 인원 제작 가능성 증가

큰 게임
→ 제작 속도 향상 가능성

공통 문제
→ 권리, 검수, 윤리, 품질 관리
```

---

## 맥락과 신뢰

AI로 누구나 무언가를 만들 수 있는 시대가 되면, 중요한 것은 단순 제작 여부가 아니라 맥락과 신뢰다.

같은 AI 결과물이라도 누가 만들었는지, 어떤 철학과 이유로 만들었는지가 중요해진다.

```text
누가 만들었는가
왜 만들었는가
어떤 철학을 가지고 있는가
어떤 맥락이 쌓였는가
```

이 맥락은 개인 크리에이터뿐 아니라 게임사에도 적용된다.

---

## 디렉터의 소통

게임 디렉터가 직접 방송에 나와 유저와 소통하는 흐름도 언급되었다.

디렉터는 이제 단순히 게임을 만드는 사람이 아니라, 유저가 게임을 믿고 시간을 투자할 수 있도록 철학과 의도를 설명하는 역할도 하게 되었다.

```text
업데이트 의도 설명
로드맵 설명
확률과 시스템 구조 설명
사과와 해명
유저와의 직접 소통
```

메이플스토리의 메이플 나우와 김창섭 디렉터의 사례가 언급되었다.

디렉터가 직접 소통하는 것은 게임 프랜차이즈 전체에 큰 영향을 줄 수 있는 자산으로 설명되었다.

---

## 게임사와 크리에이터의 거리 변화

과거에는 크리에이터가 게임사로부터 정보를 얻기 어려웠다.

지금은 홍보실이나 게임사와의 접근성이 높아졌고, 크리에이터가 유저가 원하는 정보를 더 깊게 전달할 수 있는 구조가 만들어지고 있다.

그만큼 크리에이터도 정보의 신뢰도를 높여야 한다.

```text
썸네일 어그로만으로 부정적 견해를 키우는 콘텐츠 지양
질적인 가치가 있는 정보 제공
게임사와 크리에이터의 공생 관계
```

# 6. 전체 요약

NDC 2026 3일차는 크게 세 가지 흐름으로 정리된다.

첫 번째는 **게임 데이터 분석과 AI의 결합**이다.
AI가 데이터를 분석하려면 단순한 테이블이 아니라, 맥락이 정리된 AI Ready Data가 필요하다.
NXLog, 모노레이크, 표준 로그, Semantic, Agent 구성은 AI가 신뢰 가능한 분석을 하기 위한 기반으로 소개되었다.

두 번째는 **게임사의 AX 도입 과정**이다.
넥슨과 크래프톤 모두 AI 도입에서 비슷한 고민을 하고 있었다.
효율화, 비용, 토큰 사용, 데이터 구조, 현업 확산, 워크플로우 변화, 사람의 역할 재정의가 핵심 주제로 다뤄졌다.

세 번째는 **게임과 유저를 둘러싼 연결 구조의 변화**다.
게임은 이제 단독 콘텐츠가 아니라 크리에이터, 플랫폼, 커뮤니티, IP 확장, 디렉터 소통과 함께 움직인다.
유저의 시간이 숏폼과 OTT, AI로 이동하는 시대에는 게임을 어떻게 다시 유저의 일상과 연결할 것인가가 중요한 문제로 다뤄졌다.

전체적으로 3일차의 핵심은 다음 문장으로 정리할 수 있다.

```text
AI 시대에는 데이터를 잘 쌓는 것,
업무의 본질을 다시 정의하는 것,
그리고 유저와 게임 사이의 신뢰와 맥락을 계속 연결하는 것이 중요해진다.
```
