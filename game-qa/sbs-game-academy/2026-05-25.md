# QA 수업 정리 - 데이터 테이블 검증과 기대 데미지 계산

## 1. 데이터 테이블 검증이란?

이번 수업에서는 게임 QA에서 데이터 테이블을 어떻게 검증하는지 배웠다.

게임은 굉장히 많은 데이터를 기반으로 동작한다.

예를 들어 다음과 같은 요소들이 데이터로 관리된다.

- 캐릭터별 스탯
- 직업별 스탯
- 레벨별 성장 수치
- 장비 성능
- 장비 옵션
- ENPC 스탯
- 스테이지별 보상
- 출석 보상
- 보스 패턴
- 가챠 확률
- 강화 비용
- 경험치 테이블

이처럼 게임에서 사용하는 대부분의 수치는 데이터 테이블을 통해 관리된다.

따라서 QA에서도 데이터 테이블을 기반으로 실제 게임에 적용된 값이 정상인지 검증하는 과정이 중요하다.

---

## 2. 데이터 테이블 검증이 중요한 이유

실제 게임 QA에서는 데이터 테이블, 서버, 클라이언트 사이에서 발생하는 결함이 자주 발생할 수 있다.

대표적인 결함 유형은 다음과 같다.

- 데이터 누락
- 잘못된 데이터 입력
- 서버와 클라이언트 적용값 불일치
- 데이터 테이블에는 존재하지만 게임에 적용되지 않는 경우
- 데이터 테이블에는 없는 값이 게임에 노출되는 경우
- 확률/배율/스탯 단위 해석 오류
- 데이터 타입 불일치
- 특정 조건에서 잘못된 데이터가 참조되는 경우

데이터 테이블을 제대로 활용하면 테스트 효율도 높일 수 있다.

예를 들어 캐릭터 스탯을 하나씩 게임 내에서 확인하는 것보다, 데이터 테이블의 기준값을 먼저 보고 실제 게임 적용값과 비교하는 방식이 더 효율적이다.

---

## 3. 데이터베이스와 데이터 테이블 이해

수업에서는 데이터베이스와 데이터 테이블을 도서관에 비유했다.

- 데이터베이스(DB): 도서관
- 데이터 테이블: 도서관에 비치된 책

데이터 테이블은 보통 엑셀처럼 열과 행 구조로 되어 있다.

- 열(Column): 데이터의 속성
- 행(Row): 실제 데이터 값

예시는 다음과 같다.

| 이름 | 레벨 | 공격 타입 | 공격력 | 방어력 | 체력 | 치명타 확률 | 치명타 데미지 |
|---|---:|---|---:|---:|---:|---:|---:|
| 아냐 | 30 | 근거리 | 40 | 15 | 100 | 0.15 | 1.25 |

위 표에서 `이름`, `레벨`, `공격력`, `치명타 확률` 같은 항목이 컬럼이고,  
`아냐`, `30`, `40`, `0.15` 같은 값이 실제 데이터 값이다.

## 4. 클라이언트 저장에서 서버 저장으로 바뀐 이유

과거에는 클라이언트에 데이터를 저장하는 경우가 많았지만, 요즘 네트워크 연결이 필요한 게임은 서버에 데이터를 저장하는 경우가 많다.

그 이유는 다음과 같다.

### 1. 위변조 방지

클라이언트에 중요한 데이터를 저장하면 치트엔진, 게임가디언, 트레이너 같은 도구로 조작될 가능성이 있다.

장기 서비스를 해야 하는 게임일수록 캐릭터 스탯, 재화, 장비, 확률, 보상 같은 핵심 데이터는 서버에서 관리하는 편이 안정적이다.

### 2. 작업 효율

서버에서 데이터를 관리하면 업데이트나 밸런스 수정 시 클라이언트 전체를 다시 배포하지 않고도 조정할 수 있는 경우가 있다.

### 3. 연산 효율

기기 성능에 따라 연산 처리 속도가 달라질 수 있다.

서버에서 주요 연산을 처리하고, 클라이언트는 그 결과를 받아 화면에 보여주는 방식이면 유저 기기 성능에 따른 차이를 줄일 수 있다.

## 5. 데이터 타입

데이터 테이블에서는 값의 종류에 따라 데이터 타입을 구분한다.

### int

정수형 데이터다.

예시:

- 레벨
- 수량
- 강화 단계
- 보상 개수

### float

실수형 데이터다.

예시:

- 치명타 확률
- 회피 확률
- 피해 증가율
- 배율 값

### string / varchar

문자열 데이터다.

`string`은 글자 수 제한이 없는 긴 문장에 사용될 수 있다.

예시:

- 캐릭터 설명
- 배경 설명
- 스킬 설명

`varchar`는 글자 수 제한이 있는 문자열에 사용될 수 있다.

예시:

- 유저 닉네임
- 우편 제목
- 아이템 이름

### boolean

참/거짓 데이터다.

예시:

- 출석 보상 수령 여부
- 던전 입장 조건 충족 여부
- 패스 상품 구매 여부
- 튜토리얼 완료 여부

## 6. 확률 표기와 float 값

수업 중 확률 표기와 데이터 테이블 값의 차이에 대해서도 배웠다.

유저에게 보이는 값은 보통 `%`로 표시되지만, 데이터 테이블에서는 float 값으로 저장되는 경우가 많다.

기본 변환 공식은 다음과 같다.

```text
데이터 테이블 값 = 유저 표기 확률 ÷ 100
유저 표기 확률 = 데이터 테이블 값 × 100
시는 다음과 같다.

유저에게 보이는 확률	데이터 테이블 float 값
34%	0.34
3.4%	0.034
0.34%	0.0034
0.034%	0.00034
```

따라서 0.34%를 데이터 테이블에 넣는다면 0.0034가 된다.

반대로 장비 테이블에서 CRI = 0.034처럼 % 없이 적혀 있다면, 일반적으로는 0.034%가 아니라 3.4%로 해석해야 한다.

캐릭터 테이블에서도 CRITICAL = 0.25를 25%로 해석하는 것과 같은 방식이다.

데이터 테이블을 검증할 때는 단순히 숫자가 높고 낮은지만 보는 것이 아니라, 다음 내용을 확인해야 한다.

데이터 테이블의 값과 실제 게임 적용값이 동일한가?

잘못 입력된 데이터는 없는가?
같은 규칙을 따라야 하는 데이터가 일관되게 입력되어 있는가?
캐릭터 타입과 실제 스탯 방향이 맞는가?
장비 등급과 성능이 자연스럽게 연결되는가?
확률값이 float인지 퍼센트인지 명확한가?
서버와 클라이언트가 같은 기준으로 값을 해석하는가?
테스트에 필요한 데이터 조건이 테이블에서 확인 가능한가?
실습 1. 캐릭터 50회 공격 기대 데미지 계산
계산 조건

이번 실습에서는 캐릭터가 50번 공격한다고 가정하고, 총 기대 데미지를 계산했다.

조건은 다음과 같다.

공격 횟수: 50회
치명타 확률은 절대값으로 계산
예: 치명타 확률 25%라면 10회 공격 중 2.5회는 치명타로 계산
방어력, 관통, 회피, 명중, 스킬 계수는 고려하지 않음
기본 공격력, 치명타 확률, 치명타 데미지만 사용
기대 데미지 계산 방식

처음에는 일반 공격과 치명타 공격을 나누어 계산했다.

계산 순서는 다음과 같다.

1. 치명타 횟수 = 공격 횟수 × 치명타 확률
2. 일반 공격 횟수 = 전체 공격 횟수 - 치명타 횟수
3. 일반 공격 데미지 = 공격력 × 일반 공격 횟수
4. 치명타 1회 데미지 = 공격력 × 치명타 데미지 배율
5. 치명타 총 데미지 = 치명타 1회 데미지 × 치명타 횟수
6. 총 기대 데미지 = 일반 공격 데미지 + 치명타 총 데미지

조금 더 빠르게 계산하면 다음 방식도 사용할 수 있다.

기본 데미지 = 공격력 × 공격 횟수
치명타 보너스 = 공격력 × (치명타 데미지 - 1) × 치명타 횟수
총 기대 데미지 = 기본 데미지 + 치명타 보너스

이 방식은 치명타를 “새로운 데미지”로 보는 것이 아니라,
기본 공격에 추가 데미지가 붙는 것으로 보는 방식이다.

타이거D 노장비 기대 데미지

타이거D의 기본 스탯은 다음과 같다.

항목	값
공격력	281
치명타 확률	17%
치명타 데미지	130%
1. 치명타 횟수
50 × 0.17 = 8.5회

따라서 치명타 공격은 8.5회 발생한다고 계산한다.

2. 일반 공격 횟수
50 - 8.5 = 41.5회
3. 일반 공격 데미지
281 × 41.5 = 11,661.5
4. 치명타 데미지
281 × 1.3 = 365.3
5. 치명타 총 데미지
365.3 × 8.5 = 3,105.05
6. 총 기대 데미지
11,661.5 + 3,105.05 = 14,766.55
결과
타이거D 노장비 50회 공격 총 기대 데미지 = 14,766.55
이종이 노장비 기대 데미지

이종이의 기본 스탯은 다음과 같다.

항목	값
공격력	242
치명타 확률	25%
치명타 데미지	149%
1. 치명타 횟수
50 × 0.25 = 12.5회
2. 일반 공격 횟수
50 - 12.5 = 37.5회
3. 일반 공격 데미지
242 × 37.5 = 9,075
4. 치명타 데미지
242 × 1.49 = 360.58
5. 치명타 총 데미지
360.58 × 12.5 = 4,507.25
6. 총 기대 데미지
9,075 + 4,507.25 = 13,582.25
결과
이종이 노장비 50회 공격 총 기대 데미지 = 13,582.25
노장비 기대 데미지 비교
캐릭터	공격력	치명타 확률	치명타 데미지	50회 기대 데미지
타이거D	281	17%	130%	14,766.55
이종이	242	25%	149%	13,582.25

노장비 기준에서는 타이거D가 이종이보다 기대 데미지가 높게 계산된다.

이종이는 치명타 확률과 치명타 데미지가 높지만, 타이거D의 기본 공격력이 더 높기 때문에 50회 공격 기대값에서는 타이거D가 앞선다.

피드백 후 수정한 생각

수업 중 피드백을 통해 격투가 계열의 관통과 회피 수치가 생각보다 애매하다는 점을 확인했다.

특히 격투가의 캐릭터 특색이 명확하게 드러나지 않는다는 점이 밸런스 문제로 볼 수 있었다.

예를 들어 관통형 격투가라고 해도, 관통과 회피 수치가 다른 직업군에 비해 압도적으로 높지 않다면 파티에서 해당 직업을 선택해야 하는 이유가 약해진다.

즉, “격투가가 약하다”라기보다는 다음과 같이 정리할 수 있다.

격투가의 관통/회피 특색이 수치상 명확하게 드러나지 않아,
파티 편성에서 선택 이유가 약해 보인다.
수정된 파티 방향

피드백 이후에는 손바닥 대신 땡전을 고려할 수 있다고 생각했다.

타이거D / 봉침 / 땡전
캐릭터	역할	선정 이유
타이거D	메인 딜러	높은 공격력을 기반으로 안정적인 누적 딜 기대
봉침	탱커	높은 체력/방어력/블록 기반의 전방 유지
땡전	관통형 밸런스 서브 딜러	관통 수치를 챙기면서도 체력과 방어가 크게 낮지 않음

어쌔신 계열은 딜 기대값은 높지만 후반으로 갈수록 생존 안정성이 낮아질 수 있다고 판단했다.

따라서 세 번째 자리는 단순히 딜만 보는 것이 아니라, 관통과 안정성을 함께 챙길 수 있는 방향으로 보는 것이 좋다고 생각했다.

실습 3. 장비 세팅 후 기대 데미지 계산
실습 조건

장비 테이블을 보고 타이거D와 이종이에게 기대할 수 있는 가장 높은 장비 세팅을 적용한 뒤, 최종 기대 데미지를 계산했다.

조건은 다음과 같다.

강화는 고려하지 않음
Equipment 테이블의 기본 장비 옵션만 반영
50회 공격 기준
공격력, 치명타 확률, 치명타 데미지만 우선 반영
방어력, 관통, 회피, 명중, 스킬 계수는 제외
장비를 고를 때 본 컬럼

장비 테이블에서 모든 컬럼을 다 보는 것이 아니라, 이번 계산 목적에 맞는 컬럼만 우선 확인했다.

기대 데미지 계산에서 우선적으로 본 컬럼은 다음과 같다.

컬럼	확인 이유
EquipType	무기/보조무기/장신구 구분

타이거D 강화 제외 장비 세팅

타이거D에게 적용한 장비는 다음과 같다.

타이거D 전용 무기
D등급 암살자 보조무기 CRI

장비 옵션은 다음과 같다.

장비	옵션
타이거D 전용 무기	ATK +145 / CRIDAM +0.3
D등급 암살자 보조무기 CRI	ATK +65 / CRI +0.02
최종 스탯
공격력 = 281 + 145 + 65 = 491
치명타 확률 = 0.17 + 0.02 = 0.19
치명타 데미지 = 1.3 + 0.3 = 1.6
50회 공격 계산

치명타 횟수:

50 × 0.19 = 9.5회

일반 공격 횟수:

50 - 9.5 = 40.5회

일반 공격 데미지:

491 × 40.5 = 19,885.5

치명타 1회 데미지:

491 × 1.6 = 785.6

치명타 총 데미지:

785.6 × 9.5 = 7,463.2

총 기대 데미지:

19,885.5 + 7,463.2 = 27,348.7
결과
타이거D 강화 제외 장비 세팅 후 50회 기대 데미지 = 27,348.7
이종이 강화 제외 장비 세팅

이종이에게 적용한 장비는 다음과 같다.

이종이 전용 무기
D등급 암살자 보조무기 CRD

장비 옵션은 다음과 같다.

장비	옵션
이종이 전용 무기	ATK +145 / CRI +0.15
D등급 암살자 보조무기 CRD	ATK +65 / CRIDAM +0.03
최종 스탯
공격력 = 242 + 145 + 65 = 452
치명타 확률 = 0.25 + 0.15 = 0.40
치명타 데미지 = 1.49 + 0.03 = 1.52
50회 공격 계산

치명타 횟수:

50 × 0.40 = 20회

일반 공격 횟수:

50 - 20 = 30회

일반 공격 데미지:

452 × 30 = 13,560

치명타 1회 데미지:

452 × 1.52 = 687.04

치명타 총 데미지:

687.04 × 20 = 13,740.8

총 기대 데미지:

13,560 + 13,740.8 = 27,300.8
결과
이종이 강화 제외 장비 세팅 후 50회 기대 데미지 = 27,300.8
강화 제외 장비 세팅 비교
캐릭터	최종 공격력	최종 치명타 확률	최종 치명타 데미지	50회 기대 데미지
타이거D	491	19%	160%	27,348.7
이종이	452	40%	152%	27,300.8

강화 제외 기준에서는 타이거D가 이종이보다 근소하게 높다.

27,348.7 - 27,300.8 = 47.9

두 캐릭터의 기대 데미지 차이는 크지 않다.

따라서 단순 기대 데미지만 보면 타이거D가 우세하지만, 실제 전투에서는 스킬 구조, 공격 속도, 적 방어력, 관통, 명중, 회피, 전용 효과 적용 방식에 따라 결과가 달라질 수 있다.

A등급 보조무기와 D등급 보조무기 비교

실습 중 A등급 장비가 무조건 더 좋은지에 대해서도 생각해보았다.

예를 들어 이종이에게 다음 장비를 장착한다고 가정했다.

Glove_ASS_A_CRI
A등급 암살자 보조무기
ATK +26
CRI +0.034

이 경우 이종이의 최종 치명타 확률은 다음과 같다.

기본 치명타 확률 0.25
+ 전용 무기 치명타 확률 0.15
+ A등급 보조무기 치명타 확률 0.034
= 0.434

즉, 최종 치명타 확률은 43.4%가 된다.

하지만 이 장비는 공격력 증가량이 26밖에 되지 않는다.

반면 D등급 암살자 보조무기 CRD는 ATK +65와 CRIDAM +0.03을 제공한다.

강화 제외 기준에서는 공격력 증가량 차이가 크기 때문에, A등급 CRI 보조무기보다 D등급 CRD 보조무기가 더 높은 기대 데미지를 만들 수 있다.

이 부분은 데이터 테이블 검증 관점에서 중요한 포인트라고 느꼈다.

일반적으로는 A등급 장비가 D등급 장비보다 좋아야 한다고 기대할 수 있는데, 강화 제외 기준에서는 D등급 장비의 효율이 더 높게 나타날 수 있기 때문이다.

실습 4. 강화 효율에 대한 생각

수업에서는 이후 MAX 강화 적용 시 누가 더 높은 데미지를 가지는지도 비교했다.

메모 기준으로 정리하면 다음과 같다.

노강일 때는 D등급 장비 효율이 좋음
MAX 강화 시 A등급 장비가 더 좋아짐

이를 통해 장비 성능은 기본 옵션만 보고 판단하면 안 된다는 것을 알게 되었다.

장비를 비교할 때는 다음을 함께 봐야 한다.

기본 옵션
강화 시 증가하는 스탯
최대 강화 수치
강화 비용
강화 효율
장비 등급별 성장 폭

노강 기준에서는 D등급 장비가 강해 보일 수 있지만, MAX 강화까지 고려하면 A등급 장비가 더 좋은 구조일 수 있다.

따라서 장비 밸런스를 검증할 때는 “현재 단계에서 어떤 장비가 좋은가”와 “최종 성장 기준에서 어떤 장비가 좋은가”를 나누어 확인해야 한다.

데이터 테이블을 볼 때 느낀 어려움

이번 수업에서 가장 어려웠던 부분은 데이터 테이블을 보고 빠르게 필요한 값을 식별하는 것이었다.

같은 데이터 테이블을 보고도 어떤 사람은 빠르게 깡공격력 효율이 좋다는 것을 파악하고, 어떤 장비를 껴야 기대 데미지가 높은지 계산했다.

나는 처음에 데이터 테이블의 많은 컬럼을 보면서 어떤 값을 먼저 봐야 하는지 헷갈렸다.

예를 들어 기대 데미지 계산 문제라면 HP, DEF, DODGE, BLOCK 같은 값은 일단 제외하고 ATK, CRI, CRIDAM만 봐야 한다.

하지만 처음에는 모든 컬럼이 중요해 보였고, 어떤 값을 버려도 되는지 판단하는 데 시간이 걸렸다.

이번 수업을 통해 빠른 계산은 단순히 산수를 잘하는 문제가 아니라, 다음 순서가 익숙한 사람에게 가능하다는 것을 느꼈다.

1. 문제의 목표를 먼저 확인한다.
2. 목표와 관련 없는 컬럼을 제외한다.
3. 필요한 컬럼만 남긴다.
4. 후보 장비를 추린다.
5. 계산식에 넣고 비교한다.

즉, 빠른 사람들은 모든 장비를 다 계산하는 것이 아니라, 계산할 가치가 있는 후보만 먼저 골라내고 있었다.

기대 데미지 계산을 통해 배운 점

이번 실습을 통해 공격력, 치명타 확률, 치명타 데미지가 기대 데미지에 어떻게 영향을 주는지 조금 더 이해할 수 있었다.

가장 중요한 점은 다음과 같다.

공격력은 모든 공격에 적용된다.
치명타 확률은 치명타가 발생하는 횟수에 영향을 준다.
치명타 데미지는 치명타가 발생한 공격에만 영향을 준다.

그래서 공격력 증가 옵션은 기본적으로 모든 공격에 영향을 준다.

반면 치명타 확률과 치명타 데미지는 치명타가 발생하는 부분에만 영향을 준다.

이 때문에 기대 데미지를 계산할 때 공격력 옵션이 생각보다 강하게 작용할 수 있다.

QA 관점에서의 데이터 테이블 검증 포인트

이번 수업 내용을 QA 관점으로 정리하면 다음과 같다.

1. 데이터 누락 확인

필수 컬럼에 값이 없는지 확인해야 한다.

예를 들어 캐릭터에게 필요한 공격력, 방어력, 체력, 치명타 확률 같은 값이 누락되면 게임 내 계산이 정상적으로 이루어지지 않을 수 있다.

2. 데이터 타입 확인

확률값이 float인지, 정수값인지, 문자열인지 확인해야 한다.

예를 들어 0.034가 3.4%인지 0.034%인지 기준이 명확해야 한다.

3. 데이터 규칙성 확인

같은 타입의 캐릭터나 장비는 일정한 규칙을 가져야 한다.

예를 들어 암살자 계열 장비라면 암살자 캐릭터가 착용 가능한지, 공격형 장비라면 공격력 증가가 의도대로 들어가 있는지 확인해야 한다.

4. 등급과 성능 관계 확인

A등급 장비가 D등급 장비보다 항상 좋은지, 또는 강화 전/후 기준으로 성능 우위가 바뀌는지 확인해야 한다.

이번 실습에서는 노강 기준에서는 D등급 장비가 더 좋아 보일 수 있고, MAX 강화 시 A등급 장비가 더 좋아질 수 있다는 점을 배웠다.

5. 서버와 클라이언트 적용값 확인

데이터 테이블 값이 실제 게임에 동일하게 적용되는지 확인해야 한다.

특히 확률, 배율, 강화 수치, 피해 증가 옵션은 서버와 클라이언트가 다르게 해석하면 큰 결함으로 이어질 수 있다.

6. 계산 결과와 실제 게임 결과 비교

테이블 계산값과 실제 전투 결과가 일치하는지 확인해야 한다.

예를 들어 계산상 50회 공격 기대 데미지가 특정 값으로 나왔을 때, 실제 게임에서도 같은 조건에서 유사한 결과가 나오는지 확인해야 한다.

오늘 수업을 듣고 느낀 점

이번 수업을 통해 데이터 테이블 검증은 단순히 엑셀 값을 보는 일이 아니라는 것을 느꼈다.

처음에는 데이터 테이블을 보고 숫자가 높은지 낮은지 정도만 확인하면 되는 줄 알았다.

하지만 실제로는 다음과 같은 사고 과정이 필요했다.

이 데이터가 어떤 기능에 사용되는가?
이 컬럼은 어떤 의미인가?
어떤 컬럼이 이번 테스트 목표와 직접 관련 있는가?
어떤 값은 계산에서 제외해도 되는가?
같은 규칙을 가진 데이터끼리 일관성이 있는가?
테이블 값이 실제 게임에 동일하게 적용되는가?
장비 등급과 강화 구조가 유저 기대와 맞는가?

특히 기대 데미지 계산에서 공격력, 치명타 확률, 치명타 데미지의 관계를 이해하는 데 시간이 걸렸다.

처음에는 50회 공격이라고 하면 단순히 공격력에 50을 곱하는 것부터 생각했다.

그다음 치명타 확률을 어떻게 반영해야 하는지, 치명타 데미지를 어디에 곱해야 하는지 헷갈렸다.

하지만 계산을 반복하면서 치명타 확률은 “몇 번 치명타가 발생하는지”를 결정하고, 치명타 데미지는 “치명타가 발생한 공격이 몇 배로 들어가는지”를 결정한다는 것을 이해하게 되었다.

아직 계산 속도는 느리지만, 앞으로는 다음 순서로 접근해야겠다.

1. 테스트 목표를 확인한다.
2. 필요한 컬럼만 남긴다.
3. 기본값과 장비값을 분리한다.
4. 최종 스탯을 만든다.
5. 일반 공격과 치명타 공격을 나누어 계산한다.
6. 결과를 비교한다.
7. 데이터상 이상하거나 의도 확인이 필요한 부분을 QA 관점으로 정리한다.

이번 수업은 게임 QA에서도 데이터 분석과 계산 능력이 중요하다는 것을 느낀 시간이었다.

특히 밸런스와 관련된 데이터는 단순히 감으로 판단하는 것이 아니라, 직접 계산하고 비교해야 한다는 점을 배웠다.
